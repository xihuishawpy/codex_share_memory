# Codex Memory Instructions

Use this Obsidian Codex memory vault as persistent cross-project memory.

Memory vault root:

`/Volumes/mac/my_codex_brain/Codex`

At the beginning of important work sessions, first read:

`/Volumes/mac/my_codex_brain/Codex/AGENTS.md`

When Codex learns durable background information about people, projects, decisions, follow-up items, repeatable workflows, project status, or reusable ideas, update the relevant Markdown files in this Codex memory vault.

The goal is not to save full chat history. The goal is to serialize useful memory into files that can be read, edited, compared, and reused.

Do not directly transcribe conversations. Memory updates should be concise, specific, and checkable.

Before ending important work sessions, perform a memory harvest:

- Decide whether anything from the session is worth remembering.
- If needed, update the corresponding Codex memory files.
- State which files were changed.
- Record unresolved follow-up items in `TODO.md` or `agent/open-loops.md`.

Do not store secrets, credentials, or sensitive private information.

Use this vault to keep:

- People
- Decisions
- Follow-up items
- Daily notes
- Project status
- Repeatable workflows
- Reusable ideas
- Background information that would otherwise be lost between threads

Suggested locations:

- `people/` for durable information about people.
- `projects/` for project status, decisions, and next actions.
- `agent/` for Codex operating notes and repeatable workflows.
- `notes/` for reusable ideas, daily notes, and general context.
- `TODO.md` for open follow-up items.
