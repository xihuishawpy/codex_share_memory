# Codex 记忆库

这是一个放在 Obsidian vault 里的 Codex 持久记忆库，用来保存跨会话、跨项目仍然有用的上下文。

它不是聊天记录归档，也不是自动日志系统。它的目标是把有价值的信息整理成 Markdown 文件，方便之后查询、编辑、对比和复用。

## 有什么作用

这个记忆库可以帮助 Codex 在新会话中快速恢复重要背景，例如：

- 重要人员信息
- 项目当前状态
- 已经做过的决策
- 尚未完成的待办事项
- 可复用的工作流程
- 可复用的想法和笔记
- 那些本来会在不同对话线程之间丢失的背景信息

## 目录结构

```text
Codex/
├── README.md
├── TODO.md
├── AGENTS.md
├── people/
├── projects/
├── agent/
└── notes/
```

## 各目录怎么用

- `AGENTS.md`：给 Codex 看的操作说明。新会话开始重要工作前，先让 Codex 阅读这个文件。
- `TODO.md`：记录跨会话仍然需要跟进的事项。
- `people/`：记录长期有用的人员信息。
- `projects/`：记录项目状态、决策、进展和下一步。
- `agent/`：记录 Codex 的工作规则、可复用流程、未闭环问题。
- `notes/`：记录通用笔记、想法、背景资料和每日笔记。

## 日常怎么用

开始一个重要工作会话时，可以对 Codex 说：

```text
先阅读 /Volumes/mac/my_codex_brain/Codex/AGENTS.md，然后使用这个 Codex 记忆库作为持久记忆。
```

当项目有重要进展时，可以说：

```text
把这个项目的当前状态记录到 Codex/projects/项目名.md。
```

当做出一个之后还会影响工作的决策时，可以说：

```text
把这个决策记录到 Codex 记忆库里。
```

当有跨会话待办事项时，可以说：

```text
把这个待办写入 Codex/TODO.md。
```

结束一个重要工作会话前，可以说：

```text
做一次 memory harvest，把值得长期保留的信息更新到 Codex 记忆库。
```

## 记录原则

- 只记录有长期价值的信息。
- 不保存完整聊天记录。
- 不记录密钥、凭证或敏感隐私信息。
- 记录要简洁、具体、可检查。
- 项目相关内容优先放进 `projects/`。
- 未解决事项放进 `TODO.md` 或 `agent/open-loops.md`。

## 推荐工作方式

每次让 Codex 接手一个复杂项目时，先让它阅读相关项目文件和 `AGENTS.md`。工作完成后，让它把新产生的项目状态、决策和待办更新回这个记忆库。

这样下一次换新会话时，只要重新读取这些 Markdown 文件，就能快速恢复上下文。
